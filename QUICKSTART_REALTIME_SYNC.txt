TYPICAL FLOW:

    1. User creates Detection in web UI
    2. Flask saves to PRIMARY DB (SQLite OR MySQL based on DB_TYPE)
    3. Realtime sync hook triggers
    4. Detects change = INSERT
    5. Queues sync operation
    6. Replicates to SECONDARY DB
    7. Both DBs now in sync ✓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SYNC MODES:

  sqlite_to_mysql (default)
    • Primary DB: SQLite (fast local writes)
    • Secondary: MySQL (replicated)
    • Use: Development + fallback

  mysql_to_sqlite
    • Primary DB: MySQL (central DB)
    • Secondary: SQLite (local replica)
    • Use: Production with central DB

  both
    • Both DBs written simultaneously
    • Use: Critical data (slower but guaranteed sync)

  ❌ "MySQL connection refused"

  ❌ Sync not syncing
     → Verify REALTIME_SYNC=true in .env
     → Check logs: grep "Sync" app.log
