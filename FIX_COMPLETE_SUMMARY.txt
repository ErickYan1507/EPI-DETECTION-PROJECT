## 🔧 DÉTAILS TECHNIQUES

### Qu'est-ce qui a changé dans le code?

**Avant (Ligne 124):**
```python
def _run(self):
    frame_skip = config.FRAME_SKIP
    # ...problème ici
```

**Après (Ligne 124):**
```python
def _run(self):
    global multi_detector, detector  # ← AJOUT
    
    frame_skip = config.FRAME_SKIP
    # ...maintenant ça fonctionne
```

### Pourquoi c'était nécessaire?

En Python:
- **Lecture** d'une variable globale = pas besoin de `global`
- **Modification** d'une variable globale = besoin de `global`
- **Dans un thread** qui utilise une variable globale = ⚠️ besoin de `global`

Le code accédait à `multi_detector` dans un thread séparé (`_run()`):
- Sans `global` → Python croyait avoir une variable locale
- Avec `global` → Python regarde le scope global
