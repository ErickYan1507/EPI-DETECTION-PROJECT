🎯 LA QUESTION: Comment lancer automatiquement l'Arduino physique lors de détection?
═══════════════════════════════════════════════════════════════════════════════

✅ RÉPONSE: C'est DÉJÀ AUTOMATIQUE! 

   Le système détecte une personne → Arduino s'active AUTOMATIQUEMENT
   
   Aucune manipulation manuelle ne sera nécessaire! 🎉


═══════════════════════════════════════════════════════════════════════════════
📊 FLUX COMPLET DE DÉTECTION → ACTIVATION ARDUINO
═══════════════════════════════════════════════════════════════════════════════

    ÉTAPE 1: UPLOAD IMAGE
    ┌──────────────────────────────────────┐
    │ 🖥️  Navigateur Internet              │
    │ http://127.0.0.1:5000/upload         │
    │                                      │
    │ Utilisateur sélectionne une image    │
    │ avec des personnes (EPI ou non)      │
    └──────────────────────────────────────┘
                    │
                    │ [Click → Envoyer]
                    ▼

    ÉTAPE 2: TRAITEMENT PAR FLASK
    ┌──────────────────────────────────────┐
    │ 🐍 Python Flask Application          │
    │ http://localhost:5000/api/detect     │
    │                                      │
    │ 1. Reçoit l'image                    │
    │ 2. Charge le modèle YOLO             │
    │ 3. Analyse l'image en temps réel     │
    │                                      │
    │ Détections effectuées:               │
    │   ✓ helmet (oui/non)                 │
    │   ✓ vest (oui/non)                   │
    │   ✓ glasses (oui/non)                │
    │   ✓ confidence (0-100%)              │
    └──────────────────────────────────────┘
                    │
                    │ [Résultats détection]
                    ▼

    ÉTAPE 3: CALCUL AUTOMATIQUE COMPLIANCE
    ┌──────────────────────────────────────┐
    │ 📊 ArduinoController.py              │
    │                                      │
    │ Score = (helmet + vest + glasses)    │
    │         × confidence / 100           │
    │                                      │
    │ Exemple:                             │
    │ • helmet=1, vest=1, glasses=0        │
    │ • confidence=75                      │
    │ • Score = 2 × 75 / 100 = 150%       │
    │           (normalisé à 100)          │
    └──────────────────────────────────────┘
                    │
                    │ [Score calculé]
                    ▼

    ÉTAPE 4: ENVOI AUTOMATIQUE À ARDUINO ⭐ ACTIVATION!
    ┌──────────────────────────────────────┐
    │ 📡 Commande série envoyée:           │
    │                                      │
    │ "DETECT:helmet=1,vest=1,glasses=0,   │
    │  confidence=75"                      │
    │                                      │
    │ ↓ Via port série COM3 ↓              │
    └──────────────────────────────────────┘
                    │
                    │ [Transmission série 9600 baud]
                    ▼

    ÉTAPE 5: ARDUINO REÇOIT ET TRAITE
    ┌──────────────────────────────────────────┐
    │ 🤖 Arduino MEGA 2560 (tinkercad.ino)     │
    │                                          │
    │ 1. Reçoit: DETECT:helmet=1,vest=1,...    │
    │ 2. Parse les valeurs                     │
    │ 3. Calcule le score de compliance        │
    │    Score = (1 + 1 + 0) × 75 / 100 = 150 │
    │    (Constrained to 0-100)                │
    │                                          │
    │ 4. Compare aux thresholds:               │
    │    ├─ Score >= 80 → LED VERTE ✅        │
    │    ├─ Score 60-79 → LED JAUNE  ⚠️       │
    │    └─ Score < 60  → LED ROUGE + 🔊      │
    └──────────────────────────────────────────┘
                    │
                    │ [Décision LED]
                    ▼

    ÉTAPE 6: ACTIVATION PHYSIQUE DES COMPOSANTS
    ┌───────────────────────────────────────────────────┐
    │ 💡 COMPOSANTS PHYSIQUES ACTIVÉS:                  │
    │                                                   │
    │ Cas 1: COMPLIANCE >= 80% (EPI COMPLET)            │
    │ ──────────────────────────────────────────────    │
    │ ✅ LED VERTE brille (PIN 4)                       │
    │ ⚫ LED JAUNE éteinte                              │
    │ ⚫ LED ROUGE éteinte                              │
    │ 🔇 Buzzer silencieux                             │
    │                                                   │
    │ Cas 2: COMPLIANCE 60-79% (EPI PARTIEL)            │
    │ ──────────────────────────────────────────────    │
    │ ⚫ LED VERTE éteinte                              │
    │ ⚠️  LED JAUNE brille (PIN 3)                     │
    │ ⚫ LED ROUGE éteinte                              │
    │ 🔇 Buzzer silencieux                             │
    │                                                   │
    │ Cas 3: COMPLIANCE < 60% (EPI MANQUANT)            │
    │ ──────────────────────────────────────────────    │
    │ ⚫ LED VERTE éteinte                              │
    │ ⚫ LED JAUNE éteinte                              │
    │ 🔴 LED ROUGE brille (PIN 5)                      │
    │ 🔊 Buzzer sonne: 1500Hz pendant 500ms           │
    │    puis pause 300ms, puis répète                │
    └───────────────────────────────────────────────────┘
                    │
                    │ [Activation physique visible/audible]
                    ▼

    RÉSULTAT FINAL: FEEDBACK EN TEMPS RÉEL ✨
    ┌────────────────────────────────────────┐
    │ 👀 Utilisateur observe l'Arduino       │
    │                                        │
    │ • LEDs qui s'allument automatiquement  │
    │ • Buzzer qui sonne en cas de danger    │
    │ • Tout se fait EN TEMPS RÉEL           │
    │ • Latence: <1 seconde!                 │
    └────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════════════════

  2. Cliquer: "Choisir un fichier"

  3. Sélectionner une image contenant:

⚙️  CONFIGURATION MATÉRIELLE (PINS ARDUINO)
═══════════════════════════════════════════════════════════════════════════════

  Branchements sur Arduino MEGA 2560:

  📍 PIN 2:  Capteur PIR (mouvement) - optionnel
  📍 PIN 3:  LED JAUNE (avertissement)
  📍 PIN 4:  LED VERTE (sécurité)
  📍 PIN 5:  LED ROUGE (danger)
  📍 PIN 9:  BUZZER (alerte sonore)
  📍 A0:     Capteur température - optionnel
  📍 A1:     Capteur humidité - optionnel
  📍 GND:    Masse commune
  📍 +5V:    Alimentation +5V

  Fichier concerné: tinkercad_arduino.ino (lines 32-38)

🔧 MODIFICATION DES SEUILS DE COMPLIANCE
═══════════════════════════════════════════════════════════════════════════════

  Pour changer quand les LEDs s'allument:

  Ouvrir: tinkercad_arduino.ino

  Trouver (lignes 42-44):
  ┌────────────────────────────────────────┐
  │ const int HIGH_COMPLIANCE_THRESHOLD = 80;   │  ← Seuil LED VERTE
  │ const int MEDIUM_COMPLIANCE_THRESHOLD = 60; │  ← Seuil LED JAUNE
  └────────────────────────────────────────┘

  Exemples de modification:
  
  • Rendre plus strict (exiger EPI complet):
    HIGH_COMPLIANCE_THRESHOLD = 95          (95% au lieu de 80%)
  
  • Rendre plus permissif:
    HIGH_COMPLIANCE_THRESHOLD = 70          (70% au lieu de 80%)

  Puis re-charger le sketch sur l'Arduino.

❓ QUESTIONS FRÉQUENTES
═══════════════════════════════════════════════════════════════════════════════

  Q1: "Les LEDs ne s'allument pas"
  └─ A: • Vérifier les PIN configurés correctement
        • Tester avec: Invoke-WebRequest ".../test-compliance/50"
        • Ouvrir Serial Monitor Arduino IDE pour voir les logs

  Q2: "Le buzzer ne sonne pas"
  └─ A: • Vérifier PIN 9 branché et alimenté
        • Tester avec compliance < 60%
        • Voir logs pour message "[STATUS] 🚨 DANGER"

  Q3: "Arduino pas connecté"
  └─ A: • Vérifier USB branché et reconnu sur COM3
        • Vérifier le sketch Arduino est chargé
        • Relancer avec: $env:ARDUINO_PORT = "COM3"

  Q4: "La détection ne marche pas"
  └─ A: • S'assurer que le modèle YOLO est chargé
        • Vérifier les logs: Get-Content logs/epi_detection.log
        • Tester avec une image contenant une personne claire

  Q5: "Je veux tester sans Arduino branché"
  └─ A: • Démarrer avec: $env:ARDUINO_PORT = "SIMULATION"
        • Mode simulation affiche les commandes dans les logs
        • Parfait pour déboguer sans hardware
